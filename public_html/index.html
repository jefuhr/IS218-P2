  • IMPLMENTING PYLINT
    factory is an object for creating other objects
    instantiation is the realization of a predefined object
    objects are the things you think about first in designing a program and they are also the units of code that are eventually derived from the process
    class is a blueprint for creating objects (a particular data structure), providing initial values for state (member variables or attributes), and implementations of behavior (member functions or methods).
    namespace is an object-oriented concept which allows for organization of variables, functions, and other elements of a program
    constructor is a special type of subroutine called to create an object.
    fixture provides a defined, reliable and consistent context for the tests
    type hinting means providing hints to function to only accept the given data type
    type casting is when you assign a value of one primitive data type to another type
    static means that the particular member belongs to a type itself, rather than to an instance of that type
    facade is an object that serves as a front-facing interface masking more complex underlying or structural code.
    method is a procedure associated with a message and an object.


  • AAA TESTING

    Arrange inputs and targets. Arrange steps should set up the test case.
    Act on the target behavior. Act steps should cover the main thing to be tested.
    Assert expected outcomes. Act steps should elicit some sort of response. Assert steps verify the goodness or badness of that response.

    AAA testing is simply setting up the test (arrange), testing the data set (act) and then giving a result of how good the data was in the given test (act). It simply gets more and more complex based on how complex the test is.
